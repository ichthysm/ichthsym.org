# 야간 자율 코딩

tmux 백그라운드에서 태스크별 독립 세션으로 실행합니다.
릴레이 노트로 태스크 간 맥락을 전달합니다.

## 템플릿 파일 (Progressive Disclosure)

상세 템플릿은 아래 파일을 참조하여 생성:
- **태스크 파일**: `tools/coda/templates/night-task.md.template`
- **QA 게이트**: `tools/coda/templates/night-qa-gate.md.template`
- **실행 스크립트**: `tools/coda/templates/night-run.sh.template`

## 실행 절차

### 0. 작업 주제 파싱
Arguments에서 첫 단어를 topic으로 추출 (kebab-case).
- 타임스탬프: `YYYYMMDD-HHMMSS`
- 디렉토리: `tmp/night-{topic}-{timestamp}-`

### 1. 태스크 기획

**Arguments가 있을 때**: PM 에이전트(model="opus")에게 기획 위임 → 결과를 사용자에게 보여주고 AskUserQuestion으로 승인.

**Arguments가 없을 때**: PROJECT_STATUS.md의 "다음 세션 TODO" 기반 제안 → 사용자 확인.

PM 기획 출력 형식:
```
| 번호 | 태스크명 | 작업 요약 | 의존성 |
|------|---------|----------|-------|
| 01 | kebab-case-name | 1줄 요약 | - |
```

### 2. 셋업 파일 일괄 생성

> **UI 최적화**: Task(subagent_type="general-purpose") 하나로 일괄 생성.
> 메인 화면 도구 호출: Bash(타임스탬프) 1회 + Task(파일생성) 1회 = **총 2회**

1. Bash로 타임스탬프 생성 (1회)
2. Task 서브에이전트에 모든 파일 내용을 전달하여 일괄 생성:
   - `mkdir -p {tasks_dir}`
   - 각 태스크 `.md` (night-task.md.template 기반)
   - `99-qa-gate.md` (night-qa-gate.md.template 기반)
   - `run.sh` (아래 표준 unset 블록 반드시 포함) + `chmod +x`

**run.sh 중첩 세션 방지 표준 블록 (반드시 포함):**

```bash
# 중첩 세션 방지: Claude 환경변수 전체 제거
unset CLAUDECODE 2>/dev/null || true
unset CLAUDE_CODE_ENTRYPOINT 2>/dev/null || true
unset CLAUDE_CODE_ENABLE_SDK_FILE_CHECKPOINTING 2>/dev/null || true
unset CLAUDE_SESSION_ID 2>/dev/null || true
unset CLAUDE_PROJECT_DIR 2>/dev/null || true
unset CLAUDE_CODE_SESSION_ID 2>/dev/null || true
unset CLAUDE_TOOL_CALL_ID 2>/dev/null || true
```

**claude 호출 표준 형태:**
```bash
# 올바른 형태: -p 하나만 (--print 중복 금지)
timeout 600 claude -p "$PROMPT" > "$RESULT_FILE" 2>&1
```

### 3. 상세 계획서 제출 + 최종 확인

실행 전 아래 형식의 계획서 출력 후 AskUserQuestion:

```
## 자율개발 실행 계획서
**세션**: {topic}-{timestamp} | **태스크**: {N}개 + QA 게이트
| # | 태스크 | 작업 요약 | 주요 수정 파일 | 의존성 |
```

### 4. 실행 + 보고

> **[필수] 반드시 tmux 백그라운드 세션으로 실행한다.**
> Task() 도구나 foreground 실행은 **절대 금지**.

```bash
chmod +x tmp/night-{topic}-{timestamp}-run.sh && \
tmux new -s {topic}-{short-timestamp} -d "bash tmp/night-{topic}-{timestamp}-run.sh {topic} {timestamp} 2>&1 | tee tmp/night-{topic}-{timestamp}-log.txt" && \
echo "✅ tmux 세션 시작됨" && tmux list-sessions
```

## 아침 확인 가이드

```
1. /자율결과          ← 릴레이 노트, 비용, 블로커 확인
2. (판단)             ← 결과 검토
3. /최종검수          ← 9단계 독립 재검증 + 머지 판정
```

## 함정 (Pitfalls)

- **tmux 세션 이름에 특수문자 금지** — 점(.), 콜론(:) 등은 tmux가 거부
- **headless preflight 실패 시에도 진행됨** — 태스크 실패 가능성 경고만 출력
- **CLAUDECODE 환경변수 미제거 시 중첩 세션 감지 에러** — run.sh 상단의 unset 필수. 제거 대상: `CLAUDECODE`, `CLAUDE_CODE_ENTRYPOINT`, `CLAUDE_CODE_ENABLE_SDK_FILE_CHECKPOINTING`, `CLAUDE_SESSION_ID`, `CLAUDE_PROJECT_DIR`, `CLAUDE_CODE_SESSION_ID`, `CLAUDE_TOOL_CALL_ID`
- **`claude -p --print` 중복 플래그 금지** — `-p`는 `--print`의 단축형. `claude -p "$PROMPT"` 형태만 사용할 것
- **릴레이 노트 미작성 시 다음 태스크 맥락 단절** — 태스크 프롬프트에 반드시 릴레이 필수 명시
- **on_fail: continue가 기본값** — 실패 태스크가 있어도 나머지 진행 (stop으로 변경 가능)
- **git push 절대 금지** — 캡틴 리뷰 전까지 night/ 브랜치에만 커밋

## 금지 사항
- master 브랜치에서 직접 작업
- git push (캡틴 리뷰 후)
- .env, credential 파일 수정
- docker 명령 실행
- 기존 코드 대규모 리팩토링
