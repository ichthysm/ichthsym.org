# 버전 업데이트

커밋 전 버전을 올리고 관련 파일을 동기화합니다.

## 버전 체계 (Semantic Versioning 2.0.0)

```
MAJOR.MINOR.PATCH[-PRERELEASE]
```

| 단계 | 형식 | 의미 |
|------|------|------|
| 알파 | X.Y.Z-alpha.N | 초기 개발 |
| 베타 | X.Y.Z-beta.N | 기능 완성, 안정화 중 |
| RC | X.Y.Z-rc.N | 릴리스 후보 |
| 정식 | X.Y.Z | 안정화 완료 |
| 패치 | X.Y.Z+1 | 정식 버전 버그 수정 |

## 필수 작업 순서

### 1. 현재 버전 확인
```bash
cat VERSION
```

### 2. 프리릴리스 번호 증가
```bash
# 현재 VERSION의 프리릴리스 번호를 +1
NEW_VERSION="[현재+1]"
echo "$NEW_VERSION" > VERSION
```

### 3. package.json 동기화
coda/coda.config.json의 `package_files` 목록을 참조하여 동기화:
```bash
sed -i "s/\"version\": \"[^\"]*\"/\"version\": \"$NEW_VERSION\"/" \
  package.json [추가 package.json 경로]
```

### 4. 동기화 확인
```bash
grep '"version"' package.json [추가 package.json 경로]
```

### 5. 추가 동기화 대상 확인
- Sidebar/Footer 버전 표시가 있으면 업데이트
- Swagger 버전 표시가 있으면 업데이트
- PROJECT_STATUS.md 현재 버전 필드 업데이트

### 6. 커밋 메시지 형식
```bash
git commit -m "<type>: <description> (v$NEW_VERSION)"
```

## 커밋 메시지 컨벤션
- `feat:` - 새 기능
- `fix:` - 버그 수정
- `refactor:` - 리팩토링
- `docs:` - 문서
- `chore:` - 빌드/설정
- `style:` - 디자인/스타일
- `test:` - 테스트

## 주의사항
- VERSION 파일이 단일 소스 (Single Source of Truth)
- 반드시 커밋 메시지에 `(v버전)` 태그 포함
- package.json 동기화 누락 금지
