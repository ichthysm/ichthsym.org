# 새 프로젝트 스캐폴딩

이 스킬은 새 프로젝트를 생성하고 CODA 환경을 자동으로 구성합니다.

## 단계별 진행

### 1. 프로젝트 정보 수집

사용자에게 다음 정보를 질문합니다.

```
다음 정보를 입력해주세요:

1. 프로젝트 이름: (영문, 소문자, 하이픈 허용)
2. 프로젝트 경로: (기본값: ~/othaisan/{이름})
3. 스택 선택:
   a) monorepo - NestJS + Next.js (풀스택 Monorepo)
   b) backend - NestJS만 (백엔드 전용)
   c) frontend - Next.js만 (프론트엔드 전용)
   d) minimal - Express + Vite (간단한 프로젝트)

4. 포트 번호:
   - Frontend: (기본값: 자동 제안 - 3000, 3001, 3002...)
   - Backend: (기본값: 자동 제안 - 4000, 4001, 4002...)
   - PostgreSQL: (기본값: 자동 제안 - 5432, 5433, 5434...)
   - Redis: (기본값: 자동 제안 - 6379, 6380, 6381...)

5. 프로젝트 설명: (1줄, 한글/영문)
```

**기존 포트 충돌 확인**:
- CTEM: 3000, 4000, 5432, 6379
- MsspBizCenter: 3001, 4001, 5433, 6380
- youareunderattack: 3002, 4002, 5434, 6381

다음 사용 가능 포트: 3003, 4003, 5435, 6382

## 주의사항

1. **파일 생성**: Bash tool을 사용하여 `~/.claude/` 외부 경로에 파일 생성
2. **포트 충돌**: 기존 프로젝트 포트와 자동으로 충돌 회피
3. **버전 동기화**: package.json의 version이 VERSION 파일과 일치해야 함
4. **Git 초기화**: 반드시 git init 후 .gitignore 생성
5. **스택별 맞춤**: CLAUDE.md와 package.json은 선택된 스택에 맞게 생성
6. **CODA 설정**: coda.config.json의 agents는 스택에 따라 적절히 구성
