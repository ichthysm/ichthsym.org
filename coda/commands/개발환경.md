# 개발 환경 가이드

프로젝트 개발 환경 설정 및 실행 방법입니다.

## 포트 설정

| 서비스 | 포트 | 비고 |
|--------|------|------|
| Frontend (Next.js) | 3000 | CLAUDE.md 참조 |
| Backend (NestJS) | 4000 | CLAUDE.md 참조 |
| PostgreSQL | 5432 | |
| Redis | 6379 | |

## 로컬 개발 실행

```bash
# 의존성 설치 (루트에서)
pnpm install

# 전체 개발 서버 (Turborepo)
pnpm dev

# Backend만
pnpm --filter @[scope]/backend start:dev

# Frontend만
pnpm --filter @[scope]/frontend dev
```

## Docker 실행 (권장)

```bash
cd infra/docker
cp .env.example .env
docker compose -f docker-compose.dev.yml up -d
```

**중요**: 반드시 `docker compose` (v2) 사용. `docker-compose` (v1)은 오류 발생 가능

## 주요 환경 변수

```bash
# App
PORT=4000
NODE_ENV=development

# Database (PostgreSQL)
DB_HOST=localhost
DB_PORT=5432
DB_NAME=[project_db]
DB_USERNAME=[project_user]
DB_PASSWORD=your_password

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

# JWT
JWT_SECRET=your_jwt_secret
JWT_EXPIRES_IN=3600
JWT_REFRESH_EXPIRES_IN=604800

# CORS
CORS_ORIGIN=http://localhost:3000
```

## 빌드 확인

```bash
# 전체 빌드
pnpm build

# 개별 빌드
pnpm --filter @[scope]/backend build
pnpm --filter @[scope]/frontend build
```

## URL

## 함정 (Pitfalls)

- **docker compose v2 필수** — `docker-compose` (v1)은 `ContainerConfig` 에러 발생
- **NEXT_PUBLIC_* 변경 시 프론트 재빌드 필수** — Next.js가 빌드 시점에 임베드
- **DB_SYNCHRONIZE=true는 개발환경만** — 운영에서 사용 시 데이터 손실 위험
- **pnpm install 후 node_modules 심볼릭 링크** — rm -rf node_modules 후 재설치 시 hoisted 패키지 깨짐 가능
- **포트 충돌** — 다른 프로젝트가 같은 포트를 사용 중이면 silent fail, `lsof -i :포트` 로 확인
