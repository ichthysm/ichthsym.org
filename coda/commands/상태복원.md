# 프로젝트 상태 복원

## 핵심 원칙
1. **연속성**: 이전 세션에서 중단된 작업을 바로 이어서 진행
2. **컨텍스트 복원**: 프로젝트 상태와 진행 상황을 빠르게 파악
3. **효율성**: PROJECT_STATUS.md의 참조 문서 섹션 활용

## 필수 확인 (순서대로)

### 1. 핵심 문서 읽기
1. `CLAUDE.md` - 프로젝트 컨텍스트, 개발 규칙
2. `PROJECT_STATUS.md` - 현재 상태, 진행 상황 (최근 10개 버전)
3. `VERSION` - 현재 버전

### 2. 세션 인계 정보 확인 (최우선)
PROJECT_STATUS.md의 "현재 진행 상황 (세션 인계용)" 섹션에서:
- **마지막 작업**: 이전 세션에서 수행한 작업
- **진행 중 작업**: 미완료 작업, 막힌 부분
- **다음 세션 TODO**: 이번에 해야 할 작업

### 3. 최근 변경사항 파악
- 최근 2-3개 버전의 변경사항 확인
- 현재 개발 방향 이해
- **참고**: 이전 버전 이력이 필요하면 `CHANGELOG.md` 확인

## 사용자에게 보고

다음 형식으로 간단히 보고:

```
## 프로젝트 상태 복원 완료

**프로젝트**: [이름] v[버전]
**마지막 업데이트**: [날짜]

### 이전 세션 작업
- [마지막 작업 내용]

### 미완료 작업
- [ ] [진행 중 작업]
- 막힌 부분: [있으면]

### 이번 세션 TODO
1. [우선순위 1]
2. [우선순위 2]

무엇을 먼저 진행할까요?
```

## 금지 사항
- 불필요한 코드베이스 전체 분석 금지
- 긴 요약 금지 (핵심만 간결하게)
- 체크리스트 형식 강요 금지
- PROJECT_STATUS.md에 없는 버전을 찾아 헤매지 말 것 (CHANGELOG.md 참조)

## 바로 작업 시작
사용자가 별도 지시 없으면 "다음 세션 TODO"의 첫 번째 항목부터 바로 시작
