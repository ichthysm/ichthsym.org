# Frontend 개발 가이드

프론트엔드 작업 시 참고할 규칙입니다.

## React 무한 루프/토스트 폭탄 방지

### useCallback 의존성 주의

```tsx
// 잘못된 예 - toast, t 등은 매 렌더마다 새 참조
const fetchData = useCallback(async () => {
  toast.error(t('error'));
}, [toast, t]); // 무한 루프!

// 올바른 예 - 실제 변경되는 값만 의존성에
const fetchData = useCallback(async () => {
  console.error('Failed to fetch');
}, [categoryFilter]);
```

### useEffect 의존성 주의

```tsx
// 잘못된 예 - 객체/배열 직접 의존
useEffect(() => { ... }, [{ a: 1 }]); // 매번 새 객체!

// 올바른 예 - 원시값 사용
useEffect(() => { ... }, [a]);
```

### 안정적 참조 라이브러리
