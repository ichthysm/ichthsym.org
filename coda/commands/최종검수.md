# 최종검수

자율개발(`/자율개발`) 결과물을 독립적으로 재검증합니다.
`/자율결과`로 먼저 결과를 확인한 후 실행하는 것을 권장합니다.

## 실행 절차

**Bash 도구 1회 호출**로 9단계 전체를 실행합니다.
사용자 승인 클릭 1회면 끝.

```bash
bash tools/coda/scripts/final-review.sh [SESSION_ID]
```

- SESSION_ID 생략 시 최신 릴레이 노트에서 자동 탐지
- tsc BE/FE 병렬 실행, 테스트는 OOM 방지를 위해 순차
- 결과는 스크립트가 직접 요약 출력 (PASS/FAIL/WARN/SKIP)

## 실행 후 행동

스크립트 출력 결과를 그대로 사용자에게 전달합니다.
추가 도구 호출이나 분석 없이 **스크립트 출력이 곧 보고서**입니다.

- **PASS** → "머지 가능합니다. 머지할까요?" 한 줄만 출력
- **PASS (경고 N건)** → 경고 요약 + "머지 가능합니다. 머지할까요?"
- **FAIL** → 실패 로그 경로 안내 + 수정 필요 안내

## 머지 프로세스

PASS 판정 후 사용자가 "머지"라고 하면 단일 Bash 호출:

```bash
git checkout master && \
git merge {브랜치} && \
git branch -d {브랜치} && \
rm -rf tmp/night-{SESSION_ID}-tasks/ tmp/night-{SESSION_ID}-results/ && \
rm -f tmp/night-{SESSION_ID}-relay.md tmp/night-{SESSION_ID}-blocked.md \
      tmp/night-{SESSION_ID}-log.txt tmp/night-{SESSION_ID}-run.sh && \
echo "✅ 머지 완료 + 세션 파일 정리 완료"
```

## 함정 (Pitfalls)

- **next build는 일부 타입 에러를 무시함** — tsc --noEmit을 별도 실행하는 이유
- **테스트 OOM** — BE/FE를 동시 실행하면 OOM 발생, 스크립트가 순차 실행함
- **ESLint v9 이슈** — ESLint 관련 실패는 FAIL이 아닌 SKIP 처리
- **i18n 경고는 FAIL이 아님** — 런타임 fallback 동작하므로 경고만
- **console.log 오탐** — eslint-disable, // debug 주석 라인은 자동 제외
- **API 계약 체크는 간이 비교** — 필드명 존재 여부만 확인
