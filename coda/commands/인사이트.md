# Claude Code 인사이트 (한국어)

## 핵심 원칙
1. **원본 충실**: report.html의 구조, 스타일, 레이아웃을 100% 유지
2. **완전 번역**: 요약 없이 모든 내용을 한국어로 번역
3. **기능 유지**: 복사 버튼, 타임존 선택기, 차트 등 모든 인터랙티브 기능 유지

## 실행 절차

### 1단계: 최신 리포트 생성 (headless 모드)

**중요**: 반드시 headless 모드(`claude -p`)로 실행. 대화형 CLI는 타임아웃 발생.

```bash
# headless 모드로 실행 (타임아웃 방지)
timeout 60 claude -p --print "Generate insights report" 2>/dev/null || true
```

- 성공 시 `$HOME/.claude/usage-data/report.html` 갱신
- **실패/타임아웃 시 폴백**: 기존 `report.html` 파일을 그대로 사용 (재시도 금지)

### 1.5단계: 폴백 확인

```bash
# report.html 존재 및 최근 갱신 여부 확인
ls -la $HOME/.claude/usage-data/report.html
```

- 파일이 존재하면 2단계로 진행 (생성 실패해도 기존 파일 사용)
- 파일이 없으면 사용자에게 알리고 중단

### 2단계: 원본 데이터 수집
1. `$HOME/.claude/usage-data/report.html` 읽기
2. `$HOME/.claude/usage-data/facets/*.json` 데이터 참조 (필요시)

### 3단계: 구조 그대로 번역
원본 HTML 구조를 유지하면서 텍스트만 한국어로 번역

### 4단계: 출력
`$HOME/.claude/usage-data/report-ko.html` 생성

## 번역 가이드라인

1. **기술 용어 영문 유지**:
   - TypeScript, Bash, Read, Edit, Grep, TodoWrite, Task
   - tsc --noEmit, npm test, npm run build
   - CLAUDE.md, JSON, YAML, HTML, Shell
   - Hook, Custom Skills, Task Agents

2. **코드 블록 그대로 유지**:
   - 예제 코드, 설정 파일, 명령어는 번역하지 않음
   - 복사 가능한 프롬프트 내용은 그대로 유지 (영문)

3. **자연스러운 한국어**:
   - 직역보다 의역 선호
   - 문장이 자연스럽게 읽히도록 조정

4. **숫자/통계 그대로 유지**:
   - 세션 수, 메시지 수, 퍼센트 등 원본 그대로

## 출력 확인

생성된 HTML이 다음을 만족하는지 확인:
- [ ] 모든 섹션이 원본과 동일한 순서로 포함
- [ ] 모든 차트가 동일한 데이터로 렌더링
- [ ] 복사 버튼이 정상 작동
- [ ] 인터랙티브 기능이 정상 작동

## 완료 메시지

```
## 인사이트 보고서 (한국어) 생성 완료!

한국어 보고서: file://$HOME/.claude/usage-data/report-ko.html
원본 보고서: file://$HOME/.claude/usage-data/report.html

브라우저에서 열기:
open $HOME/.claude/usage-data/report-ko.html
```

## 함정 (Pitfalls)

- **headless 모드 필수** — 대화형 `claude` CLI로 /insights 실행 시 타임아웃 발생
- **timeout 60으로 제한** — MCP 초기화 hang 방지, 실패 시 기존 report.html 폴백
- **report.html이 없으면 중단** — 한 번도 /insights를 실행한 적 없는 환경에서는 에러
- **번역 시 코드 블록 건드리지 말 것** — JavaScript/CSS 코드를 번역하면 차트 깨짐
- **파일 크기 주의** — report.html이 수 MB일 수 있음, Read 도구 한계 내에서 처리
