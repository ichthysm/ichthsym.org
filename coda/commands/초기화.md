이 코드베이스를 분석하고 CLAUDE.md 파일을 생성해주세요. 이 파일은 향후 이 저장소에서 작업할 Claude Code 인스턴스에 제공될 것입니다.

포함할 내용:
1. 빌드, 린트, 테스트 실행 방법 등 일반적으로 사용되는 명령어들. 단일 테스트 실행 방법 등 이 코드베이스에서 개발에 필요한 명령어들을 포함하세요.
2. 향후 인스턴스가 빠르게 생산적으로 작업할 수 있도록 하는 고수준 코드 아키텍처와 구조. 여러 파일을 읽어야 이해할 수 있는 "큰 그림" 아키텍처에 집중하세요.

사용 지침:
- 이미 CLAUDE.md가 있다면 개선 사항을 제안하세요.
- 초기 CLAUDE.md를 만들 때 "사용자에게 유용한 오류 메시지 제공", "모든 새 유틸리티에 대한 단위 테스트 작성", "코드나 커밋에 민감한 정보(API 키, 토큰) 포함 금지" 같은 당연한 지침은 포함하지 마세요.
- 쉽게 발견할 수 있는 모든 컴포넌트나 파일 구조를 나열하지 마세요.
- 일반적인 개발 관행은 포함하지 마세요.
- Cursor 규칙(.cursor/rules/ 또는 .cursorrules)이나 Copilot 규칙(.github/copilot-instructions.md)이 있다면 중요한 부분을 포함하세요.
- README.md가 있다면 중요한 부분을 포함하세요.
- 다른 파일에 명시적으로 포함되어 있지 않는 한 "일반적인 개발 작업", "개발 팁", "지원 및 문서" 같은 정보를 만들어내지 마세요.
- 파일 시작 부분에 다음 텍스트를 반드시 포함하세요:

```
# CLAUDE.md

이 파일은 이 저장소에서 코드 작업을 할 때 Claude Code (claude.ai/code)에게 가이드를 제공합니다.
```
